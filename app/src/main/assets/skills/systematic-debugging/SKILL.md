# 系统化调试

## 核心规则

**没有根因分析，就不准修bug。**

## 调试流程

### 1. 收集证据
- 重现bug（确认真实存在）
- 记录完整错误信息（stacktrace/日志/状态）
- 缩小范围（哪个模块、哪行代码）

### 2. 根因分析
- 问"为什么"至少3次
- 区分症状和根因
- 确认根因（不是猜测）

### 3. 最小修复
- 只修根因，不修症状
- 最小改动原则
- 添加测试防止复发

### 4. 验证修复
- 运行原有测试
- 添加新的针对性测试
- 确认bug确实消失

## 反模式(禁止)

- 看到错误就猜原因 → 必须分析
- 修症状不修根因 → 必须找到根因
- "可能是..." → 必须确认
- 随意添加try-catch → 必须理解为什么出错
