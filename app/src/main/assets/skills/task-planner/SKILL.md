# Task Planner - 任务分解与规划技能

## 概述

task-planner 是一个强大的任务分解与规划技能，专门用于将复杂任务分解为可执行的子任务，管理任务间的依赖关系，并制定并行执行策略以优化整体执行效率。

## 核心能力

### 1. 任务分解
- 将复杂的大任务分解为多个独立的子任务
- 识别子任务之间的依赖关系
- 为每个子任务分配适当的优先级

### 2. 依赖管理
- 支持父子任务依赖关系（`kanban_link`）
- 自动识别任务间的前置依赖
- 构建任务依赖图

### 3. 并行执行策略
- 识别可并行执行的独立任务
- 优化任务执行顺序
- 最大化并行度以提高效率

### 4. 任务状态跟踪
- 创建看板任务（`kanban_create`）
- 更新任务状态（`kanban_complete`）
- 标记阻塞状态（`kanban_block`）
- 发送心跳保持任务活跃（`kanban_heartbeat`）

## 使用流程

### 第一步：分析任务
1. 理解目标任务的复杂度和范围
2. 识别关键子任务和里程碑
3. 确定任务间的依赖关系

### 第二步：创建任务结构
```
# 创建父任务
kanban_create(title="主任务标题", body="详细描述", priority=8)

# 创建子任务
kanban_create(title="子任务1", body="描述", parents=["父任务ID"])
kanban_create(title="子任务2", body="描述", parents=["父任务ID"])

# 建立依赖关系
kanban_link(parent_id="前置任务ID", child_id="后置任务ID")
```

### 第三步：并行执行
- 同时执行没有依赖关系的子任务
- 使用 `kanban_heartbeat` 保持长任务活跃
- 遇到阻塞时使用 `kanban_block` 标记

### 第四步：完成任务
- 使用 `kanban_complete` 标记子任务完成
- 附带元数据记录结果
- 更新黑板状态

## 最佳实践

### 任务分解原则
1. **原子性**：每个子任务应该是独立的、可执行的单元
2. **可衡量**：每个任务有明确的完成标准
3. **粒度适中**：避免过细或过粗的分解
4. **优先级明确**：根据重要性和紧急性设置优先级

### 依赖管理原则
1. 尽量减少不必要的依赖以提高并行度
2. 识别关键路径（最长的依赖链）
3. 对于可选依赖，使用 `kanban_link` 明确标注

### 并行执行策略
1. 识别可并行的任务组（无相互依赖）
2. 为每个并行组分配独立的工作单元
3. 监控并行任务的执行状态

## 可用工具

| 工具 | 用途 |
|------|------|
| `kanban_create` | 创建新任务，支持父任务和技能指定 |
| `kanban_show` | 查看任务详情，包括描述、状态和事件日志 |
| `kanban_link` | 建立任务间的依赖关系 |
| `kanban_complete` | 标记任务完成，附带元数据 |
| `kanban_block` | 标记任务阻塞，说明阻塞原因 |
| `kanban_comment` | 添加评论或更新黑板状态 |
| `kanban_heartbeat` | 发送心跳，防止任务被回收 |

## 示例场景

### 场景1：多步骤数据处理
```
1. 创建"数据管道构建"父任务
2. 创建子任务：
   - "数据源接入"（优先级8）
   - "数据清洗"（依赖：数据源接入）
   - "数据转换"（依赖：数据清洗）
   - "数据验证"（依赖：数据清洗，可并行于数据转换）
   - "数据输出"（依赖：数据转换、数据验证）
```

### 场景2：软件开发任务
```
1. 创建"功能开发"父任务
2. 创建子任务：
   - "需求分析"（优先级10）
   - "架构设计"（依赖：需求分析）
   - "前端开发"（依赖：架构设计）
   - "后端开发"（依赖：架构设计，可并行于前端开发）
   - "集成测试"（依赖：前端开发、后端开发）
   - "部署上线"（依赖：集成测试）
```

## 注意事项

1. **任务ID管理**：创建任务后保存返回的任务ID，用于后续操作
2. **优先级设置**：1-10，10为最高优先级
3. **阻塞处理**：遇到无法继续的情况，使用 `kanban_block` 而非无限等待
4. **心跳维护**：长时间运行的任务应定期调用 `kanban_heartbeat`
5. **元数据记录**：完成任务时记录关键结果和状态
