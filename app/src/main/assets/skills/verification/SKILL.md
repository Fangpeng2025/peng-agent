# 完成前验证

## 核心规则

**没有验证证据，就不准声称完成。**

## 验证清单

1. **功能验证**: 实际运行代码/命令，确认输出正确
2. **边界验证**: 空输入、超长输入、非法输入
3. **集成验证**: 与其他模块交互正常
4. **回归验证**: 之前的功能没被破坏

## 验证方式

- 代码: 运行测试 `pytest`/`cargo test`
- 构建: 运行构建 `./gradlew assembleDebug`
- 功能: 实际执行命令并检查输出
- 文件: `stat`/`ls`确认文件存在

## 反模式(禁止)

- "代码已写完" (没有运行过)
- "应该没问题" (没有验证)
- "和之前类似" (没有实际测试)
- 只写stub/placeholder，声称完成
